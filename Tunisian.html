Why This Project

I built the project with the intention of predicting the house prices of new zealand. However, I was not able to find a dataset that is suitable for the project. I then decided to use a dataset from Kaggle that contains house prices in Tunisia. I was able to find a dataset that contains 80 features and 6000+ rows. I then decided to use the dataset to predict the house prices in Tunisia.

Feature Engineering

  • Dataset: Tunisian House Price
  • Technical Implementation: Upon exploring the dataset, I was met with 25 features which contained number of bedrooms, bathrooms, and other features. I then decided to use the following features to predict the house prices:
    • price_eur
    • location
    • Area
    • room
    • bathroom
    • age
    • state
    • distance_to_capital
    • garage

Classification Model Implementation

Technical Implementation: I made a price per meter squared by dividing the price of the unit by the area of the unit. After then, I sorted the data by location and filtered by how many units per area. Areas with less than 10 properties where removed from the dataset. Using the standard deviation of the price per m2 calculated, I was able to remove the outliers in the dataset. I then used some visualisation such as heatmaps to see the relationship of the features that remained. Lastly, I used the following models to predict the house prices:
  • Linear Regression
  • Lasso
  • Decision Tree

Evaluation of the Implemented Program

I have compared the results that I got using the model that I made. However, the difference from the tunisian real estate website and the model that I made is quite big. I think that the reason for this is that the dataset that I used is not updated. I also think that I missed other features such economic status of the country, and other features that are not included in the dataset. I think that the model that I made is not suitable for predicting the house prices in Tunisia.
